EKS Lifecycle & Model Serving

A comprehensive guide to Amazon EKS: cluster lifecycle, integration with AWS services, and a full hands-on walkthrough to deploy a machine learning inference service.

The EKS lifecycle

Amazon EKS provides a robust, repeatable framework for managing the entire lifecycle of containerized workloads — from cluster creation to model serving and scaling.

1. Cluster creation

Provision the Kubernetes control plane and compute fabric. Use eksctl, the official CLI:

$ eksctl create cluster --name demo-cluster --region us-east-1 --nodegroup-name demo-nodes --node-type t3.small --nodes 2
  • Multi-AZ control plane — managed by AWS across 3 AZs.
  • Managed node group — EC2 instances (t3.small) provisioned and registered.
  • Networking — VPC, subnets, security groups, route tables created automatically.
  • IAM roles — for control plane and node group.
  • Add-ons — CoreDNS, kube-proxy, VPC CNI installed by default.

2. Node registration

Each EC2 instance boots from the EKS-optimized AMI and runs a bootstrap script. The IAM role is mapped to Kubernetes node identity via the aws-auth ConfigMap in the kube-system namespace. Nodes become available in the scheduling pool.

Key takeaway: Nodes are standard EC2 instances, but once joined to EKS they behave as part of the Kubernetes cluster — AWS-specific steps are abstracted away.

3. Deploying workloads

Use standard Kubernetes manifests (Deployment, Service). Integration with AWS services is seamless:

  • LoadBalancer — provisions an AWS NLB/ALB (or CLB) automatically.
  • Persistent volumes — via EBS CSI driver for stateful workloads.
  • S3 access — for large datasets or model artifacts.

4. Scaling & updates

Cluster Autoscaler
Watches for unschedulable pods and triggers Auto Scaling Groups to add/remove EC2 nodes.
Horizontal Pod Autoscaler
Scales replicas of your model-serving container based on metrics like latency or CPU.
Control plane upgrades
AWS manages the control plane; you initiate version upgrades via CLI/console.
Add-on updates
CoreDNS, kube-proxy, VPC CNI — update with version compatibility guarantees.

Integration into the AWS ecosystem

EKS deeply integrates with AWS services, making it a unified cloud operations platform.

🔐 IAM
IAM roles mapped to Kubernetes via aws-auth. IRSA (IAM Roles for Service Accounts) allows per-pod permissions to S3, DynamoDB, etc.
🌐 Networking
VPC CNI assigns VPC IPs to pods. Route 53 for DNS, AWS Load Balancer Controller for Ingress, VPN for hybrid architectures.
💾 Storage
EBS (block), S3 (object), EFS (file) via CSI drivers. Root volumes for nodes are EBS by default.
🛡️ Security
KMS for secrets, VPC security groups, WAF + ALB for DDoS protection, and CloudTrail auditing.
📊 Observability
CloudWatch, Fluent Bit, and Prometheus metrics. Custom metrics from model-serving containers.

Design & operational considerations

Cluster topology
Control plane across multiple AZs. Spread node groups across AZs for fault tolerance. Use managed node groups for simpler lifecycle.
Networking strategy
Public vs private endpoints. Control plane in private subnets for internal traffic. Optimize for low latency — same region as users.
Storage & data
EBS for fast block access (model cache), S3 for durable artifact storage. Choose based on workload.
Cost optimization
Use node group autoscaling to scale down during off-peak. Right-size instance types (e.g., t3.small for inference). Leverage Savings Plans.
Observability
Collect logs, metrics, traces. Create CloudWatch dashboards and alerts for pod restarts, latency spikes, and node health.

Setting up an AWS Free Tier account

1
Prerequisites: unique email, mobile phone, valid credit/debit card (for identity verification, no charges for Free Tier).
2
Sign up: go to aws.amazon.com/free → click "Create a Free Account". Verify email, set root password, choose Free plan.
3
Contact & payment: provide personal/business info, add payment method (temporary auth hold).
4
Identity verification: SMS or voice call, enter code. Choose Basic Support (free).
5
Activation: wait for "Welcome" email (minutes to 24 hours). Log in as root user.
6
Secure root: enable MFA (authenticator app recommended). Create an IAM admin user (e.g., admin) with AdministratorAccess policy. Generate access keys for CLI.
Never use root access keys. Always use IAM admin user for daily work. Download and store credentials securely.

Installing AWS CLI & eksctl

AWS CLI — install via pip install awscli or package manager. Configure with:

$ aws configure
AWS Access Key ID: AKIA...
AWS Secret Access Key: wJalr...
Default region name: us-east-1
Default output format: json

eksctl — official CLI for EKS. Install from GitHub releases or package managers. Verify:

$ eksctl version
0.215.0

kubectl — ensure it's installed (v1.32+ recommended).

Hands-on demo: deploying an ML model on EKS

Project structure

├── app.py # FastAPI inference app
├── train.py # train simple model (y=2x)
├── model.pkl # saved model
├── requirements.txt # dependencies
├── Dockerfile # container definition
├── deployment.yaml # K8s Deployment
└── service.yaml # K8s Service (LoadBalancer)

1. Train & save model

import numpy as np
from sklearn.linear_model import LinearRegression
import joblib
X = np.array([[1],[2],[3],[4],[5]])
y = np.array([2,4,6,8,10])
model = LinearRegression().fit(X, y)
joblib.dump(model, 'model.pkl')

2. FastAPI inference app

from fastapi import FastAPI
import joblib, numpy as np
app = FastAPI()
model = joblib.load('model.pkl')
@app.post("/predict")
def predict(x: float):
return {"prediction": model.predict([[x]])[0]}

3. Containerization

FROM python:3.12-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "80"]

Build and push to Docker Hub:

$ docker build -t yourusername/aws-demo:v1 .
$ docker push yourusername/aws-demo:v1

4. Kubernetes manifests

deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata: name: k-demo-dep
spec:
replicas: 2
selector: matchLabels: app: k-demo
template:
metadata: labels: app: k-demo
spec: containers:
- name: k-demo
image: yourusername/aws-demo:v1
ports: - containerPort: 80

service.yaml

apiVersion: v1
kind: Service
metadata: name: k-demo
spec:
selector: app: k-demo
ports: - port: 80 targetPort: 80
type: LoadBalancer

5. Create EKS cluster

$ eksctl create cluster --name demo-cluster --region us-east-1 --nodegroup-name demo-nodes --node-type t3.small --nodes 2

⏱️ Takes ~15-20 minutes. Ensure Free Tier instance type (t3.small) is available.

6. Deploy to EKS

$ kubectl apply -f deployment.yaml
$ kubectl apply -f service.yaml
$ kubectl get pods
$ kubectl get svc # note the EXTERNAL-IP

7. Test the API

import requests
response = requests.post("http://<EXTERNAL-IP>/predict", json={"x": 7.5})
print(response.json()) # {"prediction": 15.0}

FastAPI interactive docs: http://<EXTERNAL-IP>/docs

8. Clean up

$ kubectl delete svc k-demo
$ kubectl delete deploy k-demo-dep
$ eksctl delete cluster --name demo-cluster
Always clean up to avoid unexpected charges. Verify in the AWS Console that no resources remain.
